# ESP32 BLE IMU Session Controller

Arduino sketch for a Seeed Studio XIAO ESP32S3 with an ICM-20948 IMU. The
ESP32 advertises over BLE, sends a gesture menu to the mobile app, waits for a
task selection, collects IMU samples only during that task, runs the Edge
Impulse classifier locally, and notifies the app with session progress and
results.

## Hardware

Default I2C pins match the existing XIAO ESP32S3 pin layout:

| ICM-20948 | XIAO ESP32S3 |
| --- | --- |
| SDA | D4 / GPIO5 |
| SCL | D5 / GPIO6 |
| VIN / VCC | 3V3 |
| GND | GND |

Override the pins by editing the fallback values near the top of the sketch, or
by passing equivalent build flags:

```cpp
#define IMU_I2C_SDA 5
#define IMU_I2C_SCL 6
```

## BLE Contract

The sketch advertises as `IMU-Datastream` and uses the Nordic UART Service
layout:

| Direction | UUID |
| --- | --- |
| Service | `6E400001-B5A3-F393-E0A9-E50E24DCCA9E` |
| RX write | `6E400002-B5A3-F393-E0A9-E50E24DCCA9E` |
| TX notify | `6E400003-B5A3-F393-E0A9-E50E24DCCA9E` |

Notifications are UTF-8 JSON lines split into 20-byte BLE-safe chunks. Rejoin
chunks until `\n` to recover each full JSON message.

The phone does not run inference and does not look for a raw sensor stream. It
connects to the ESP32, receives the gesture menu, writes a task selection, and
displays the JSON events/results generated by the ESP32.

## Commands

Write one of these commands to the RX characteristic:

| Command | Behavior |
| --- | --- |
| `1` to `6` | Start a 10-repetition session for the selected gesture |
| `m`, `h`, or `?` | Send the gesture menu again |
| `r` | Reset/reinitialize the IMU |
| `x` | Request the current session to stop |

The sketch sends the menu when it starts and again when a BLE central connects.
It does not collect IMU samples continuously. IMU reads only occur inside the
selected movement window after a valid task command is received.

## Gesture Menu

```json
{
  "type": "gesture_menu",
  "device_id": "xiao_esp32s3_ble_001",
  "prompt": "select_gesture_1_to_6",
  "gestures": [
    {"selection": 1, "label": "Flexion"},
    {"selection": 2, "label": "Extension"}
  ]
}
```

The current sketch exposes six gestures:

| Selection | Gesture |
| --- | --- |
| `1` | Flexion |
| `2` | Extension |
| `3` | Pronation |
| `4` | Supination |
| `5` | Radial Deviation |
| `6` | Ulnar Deviation |

## Session Payloads

During a task session the ESP32 notifies:

| Type | Purpose |
| --- | --- |
| `session_start` | Selected target gesture and repetition count |
| `repetition_event` | Countdown, movement start, sampling finished, and next-rep status |
| `inference_result` | On-device Edge Impulse result for one repetition |
| `session_summary` | Accuracy/pass-rate summary for completed repetitions |
| `status` | Startup, reset, stop, and error messages |

Example inference result:

```json
{
  "type": "inference_result",
  "device_id": "xiao_esp32s3_ble_001",
  "repetition": 1,
  "window_id": 1,
  "target": "Flexion",
  "ok": true,
  "predicted": "Flexion",
  "label": "Flexion",
  "correct": true,
  "trusted": true,
  "confidence": 0.94,
  "scores": {"Extension": 0.01, "Flexion": 0.94},
  "sample_count": 33,
  "axes_per_sample": 6,
  "input_frame_size": 198,
  "sample_interval_ms": 60.606061,
  "collect_ms": 2000.0,
  "inference_ms": 3.1,
  "timing_ms": {"wall": 3.1, "dsp": 0, "classification": 3, "anomaly": 0},
  "error": null
}
```

Raw IMU features are not sent over BLE in the normal path.

The Edge Impulse Arduino library that provides `ei_gesture_left_hand_imu.h` must
be installed. The sketch uses its window size, sampling interval, and classifier
label macros.
